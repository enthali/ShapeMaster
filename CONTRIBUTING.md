# Contributing to ShapeMaster

Thank you for your interest in contributing to ShapeMaster! We welcome contributions from the community. Please follow the guidelines below to get started.

## How to Contribute



1. **Fork the Repository**
   - Click the "Fork" button at the top right of the GitHub page to create your own copy of the repository.

2. **Clone Your Fork**
   - Clone your forked repository to your local machine:
     ```pwsh
     git clone https://github.com/your-username/ShapeMaster.git
     ```

3. **Create a Branch**
   - Create a new branch for your feature or bugfix:
     ```pwsh
     git checkout -b my-feature-branch
     ```

4. **Make Your Changes**
   - Implement your feature or fix the bug.
   - Please follow the existing code style and add comments where appropriate.

5. **Test Your Changes**
   - Ensure the project builds and runs as expected.
   - If possible, add or update tests.

6. **Commit and Push**
   - Commit your changes with a clear message:
     ```pwsh
     git add .
     git commit -m "Describe your change"
     git push origin my-feature-branch
     ```

7. **Open a Pull Request**
   - Go to the original ShapeMaster repository on GitHub.
   - Click "Compare & pull request" and submit your PR.
   - Describe your changes and reference any related issues.

8. **Review Process**
   - The project maintainer will review your PR.
   - You may be asked to make changes before it is merged.

## Developer Setup

For detailed setup instructions, see [InstallForDevelopers.md](InstallForDevelopers.md).

## Code of Conduct

Please be respectful and constructive in all interactions.

> ## Why fork?
> Forking creates your own copy of the repository on GitHub. This is the standard open source workflow: you make changes in your fork, then submit a pull request. This keeps the main project secure and lets you propose changes without needing direct access to the main repository.
